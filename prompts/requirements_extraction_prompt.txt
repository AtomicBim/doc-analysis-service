Проанализируй следующий текст технического задания (ТЗ).

Твоя задача — извлечь **все содержательные требования** и вывести их в строго заданном формате.

**Обязательные правила обработки:**

1. **Игнорируй всё, что не является содержательным требованием**:
   - Шапки, колонтитулы, логотипы, водяные знаки, подписи, даты, ФИО, должности, реквизиты.
   - Служебные надписи: «утверждено», «согласовано», «разработано» и т.п.
   - Повторяющиеся элементы оформления (название проекта на каждой странице и др.).

2. **Работай с таблицами корректно**:
   - Если в ячейке перечислены несколько элементов через запятую, точку с запятой или перенос строки — **раздели их на отдельные требования**, сохраняя общий контекст из других ячеек той же строки.
   - При объединении ячеек одной строки — соединяй их через символ ` | ` (пробел-вертикальная черта-пробел).
   - Пример:  
     Строка: «Тип помещения: выставочный зал, кофейня» →  
     Два требования:  
     `Выставочный зал | Тип помещения: выставочный зал`  
     `Кофейня | Тип помещения: кофейня`

3. **Формулировки**:
   - Сохраняй точность: если в документе «не менее 3,5 м» — не меняй на «минимум».
   - Все требования должны быть выражены в нормативной форме: «Предусмотреть…», «Обеспечить…», «Высота должна быть не менее…» и т.п.
   - Не добавляй интерпретаций, пояснений или обобщений.

4. **ФОРМАТ ВЫВОДА (строго соблюдать):**  
   Верни JSON объект с полем `"requirements"`, содержащим массив требований.
   
   Каждое требование должно иметь поля:
   - `"number"`: порядковый номер требования (целое число, начиная с 1)
   - `"text"`: полный текст требования. Если исходно было в таблице — объедини все релевантные ячейки через ` | `
   - `"section"`: название раздела или категории (например: «Общие характеристики объекта», «Квартирография», «Коммерческая недвижимость», «Инженерные системы», «Благоустройство», «Служебные помещения» и т.д.). Определи раздел по контексту расположения требования в документе.
   - `"trace_id"`: уникальный идентификатор в формате `"req-{number}"` (например: `"req-1"`)

5. **Пример формата ответа**:
```json
{
  "requirements": [
    {
      "number": 1,
      "text": "Высота потолков должна быть не менее 2.64 м",
      "section": "Общие характеристики объекта",
      "trace_id": "req-1"
    },
    {
      "number": 2,
      "text": "Предусмотреть паркинг на 50 машиномест",
      "section": "Инфраструктура",
      "trace_id": "req-2"
    }
  ]
}
```

Верни **строго в этом JSON формате**, без дополнительных пояснений.
